# Federation Topology

Status: canonical doctrine draft  
Scope: conceptual and software architecture for self-folding Federation systems

## Purpose

This document defines the topology that guides AI Freedom Trust Federation interfaces, repositories, trust seeds, and agentic systems.

The topology is a design model, not an empirical physics claim. It is used to organize software, governance, knowledge, maps, and contribution flows.

## Core Model

The Federation is modeled as a self-folding manifold.

Every user, trust, repository, and community experiences the system from its own Green Zone: the local present layer where observation, identity, context, and action meet.

From that center, the system unfolds into three temporal directions:

- Past / Redshift: history, records, receipts, commits, releases, lessons, lineage, and memory.
- Present / Greenshift: current location, active tasks, working branch, open decisions, permissions, and live context.
- Future / Blueshift: proposals, pull requests, roadmaps, goals, milestones, possible designs, and attractors.

When differentiated work becomes stable, it reaches a coherence point: a release, constitutional amendment, accepted proposal, completed service, verified public good, or shared standard.

## Software Interpretation

The topology becomes practical in software as:

- a map centered on the user's context;
- a Git graph centered on the current branch;
- a project graph centered on active work;
- an AI memory graph centered on permitted local knowledge;
- a trust graph centered on consent and stewardship;
- a marketplace graph centered on productive value;
- a governance graph centered on current decisions.

## Light Cone Interpretation

The familiar light-cone diagram is treated as a local cross-section of a larger closed circulation.

In software terms:

- the past branch stores what has already happened;
- the future branch stores what may become real;
- the present membrane is where decisions and observations occur;
- coherence points are where many possible paths crystallize into one stable artifact.

## Torus Interpretation

The torus represents recirculation, not decoration.

Records from the past inform present decisions. Present decisions shape future possibilities. Future goals reorganize present priorities. Completed future work becomes new history.

The loop is continuous.

## Holographic Recursion

Every level should preserve the same pattern:

- Federation;
- community;
- trust;
- project;
- repository;
- document;
- section;
- task;
- contribution.

Each level has identity, context, history, present state, future path, governance, and contribution records.

## Interface Rule

The interface should not feel like unrelated pages.

It should feel like zooming through one coherent field:

- Earth to region;
- region to community;
- community to trust;
- trust to project;
- project to repository;
- repository to document;
- document to task;
- task to contribution.

## Agent Rule

AI agents should reason topologically:

1. Identify the current Green Zone.
2. Inspect the past record.
3. Name the present decision.
4. Identify future branches.
5. Preserve evidence boundaries.
6. Suggest the next coherent action.
7. Report what changed and what remains uncertain.

## Constitutional Rule

No topology may erase local sovereignty.

Every node may be part of a larger field, but each trust remains responsible for its own consent, permissions, governance, and disclosure boundaries.
