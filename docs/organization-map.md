# AI Freedom Trust Federation Organization Map

AI Freedom Trust Federation should be organized as an umbrella with clear repository roles, maturity levels, and public expectations.

## Umbrella Layers

Purpose: define the public mission, governance posture, repository standards, and visible project map.

Primary repository:

- `www.aifreedomtrust.com` - canonical public website and public-facing organization portal

Supporting repository:

- `AIFT-Forge` - profile, templates, doctrine, coordination patterns, and shared operating language

## Repository Status Labels

Use these labels consistently in READMEs and reports:

- `Active product` - actively maintained and public-facing
- `Infrastructure` - foundational platform or deployment tooling
- `Federation core` - shared doctrine, templates, coordination, or public identity
- `Research` - experimental, model, dataset, cryptographic, or protocol work
- `Incubating` - promising but not yet a primary public surface
- `Legacy` - retained for history, dependency, or reference
- `Fork` - external code fork that should not be confused with original AIFT work
